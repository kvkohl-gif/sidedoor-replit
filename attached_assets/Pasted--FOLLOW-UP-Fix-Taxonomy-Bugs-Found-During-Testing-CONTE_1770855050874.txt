# FOLLOW-UP: Fix Taxonomy Bugs Found During Testing

## CONTEXT
We just ran 20 real-world job titles through the role taxonomy system you implemented. It caught 90% of titles correctly, but we found 2 seniority detection bugs and 1 gap in hiring manager targets. These need to be fixed before launch.

**Do NOT re-implement the taxonomy system. It's already in place. Only make the targeted fixes below.**

---

## BUG FIX 1: "Chief of Staff" incorrectly detected as C-Suite seniority

**Problem:** The word "chief" in "Chief of Staff" triggers the c_suite seniority pattern. But Chief of Staff is a senior manager / operations role, NOT a C-suite executive.

**File to edit:** `backend/roleTaxonomyService.ts`

**Fix:** In the `detectSeniority()` function, add an explicit exception check BEFORE the main seniority loop. Add this right after the `const normalizedTitle` line and before the `const orderedKeys` line:

```typescript
// --- EXCEPTIONS: titles that contain seniority keywords but aren't that level ---
const seniorityExceptions: Record<string, SeniorityResult> = {
  'chief of staff': { seniority: 'senior_manager', level: 6, apolloSeniority: 'director', confidence: 0.9 },
};

for (const [exceptionPattern, exceptionResult] of Object.entries(seniorityExceptions)) {
  if (normalizedTitle.includes(exceptionPattern)) {
    return exceptionResult;
  }
}
// --- END EXCEPTIONS ---
```

## BUG FIX 2: "partner" pattern in c_suite is too broad

**Problem:** The c_suite seniority pattern includes "partner" which matches "HR Business Partner", "Talent Partner", "People Partner", "Client Partner" — all mid-level roles. It should only match executive-level partner titles like "Managing Partner" or "General Partner".

**File to edit:** `backend/data/roleTaxonomy.json`

**Fix:** In the `seniorityLevels.c_suite.patterns` array, replace `"partner"` with more specific patterns:

**Before:**
```json
"c_suite": {
  "level": 9,
  "patterns": ["chief", "c-level", "cto", "cfo", "cmo", "coo", "cpo", "cro", "ceo", "founder", "co.?founder", "partner"],
  "apolloSeniority": "c_suite"
}
```

**After:**
```json
"c_suite": {
  "level": 9,
  "patterns": ["chief", "c-level", "cto", "cfo", "cmo", "coo", "cpo", "cro", "ceo", "founder", "co.?founder", "managing partner", "general partner", "founding partner", "senior partner"],
  "apolloSeniority": "c_suite"
}
```

This way "HR Business Partner" and "Talent Partner" won't match c_suite, but "Managing Partner" at a law firm or consulting firm still will.
