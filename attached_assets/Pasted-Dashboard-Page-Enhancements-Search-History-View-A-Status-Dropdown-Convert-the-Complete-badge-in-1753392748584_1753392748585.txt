Job Detail Page Enhancements (Results View)

👤 Per Contact (Recruiter Card) Enhancements
1. Status Dropdown per Contact
* Replace "Valid/Verified" badge or add an additional dropdown.
* Options:
    * Not Contacted
    * Email Sent
    * Awaiting Reply
    * Follow-Up Needed
    * Rejected
    * Interview Scheduled
* Each status should have a color-coded icon/dot for quick visual reference.
* Default is Not Contacted.
2. Last Action Metadata
* Add metadata beneath the contact name or below the status:
    * Example: Last contacted: Jul 23
* Auto-filled when user sends an email via platform.
* Allow manual editing via pencil icon or hover-edit.
3. Contacted / Not Contacted Filter
* Add a toggle/filter bar at the top of the recruiter list:
    * Show: All | Contacted | Not Contacted
* Based on the current status field — Not Contacted = default; all others = Contacted.
4. Inline Notes per Contact
* Add small ✏️ icon or “Add Note” link.
* Inline textarea expands when clicked.
* Example note: “Met at Women in Tech NYC – circle back post-August.”
* Save per contact and persist in backend.
5. Generate Email & Message Editor
* For each recruiter card:
    * Add a “Generate Email” button.
    * Clicking expands a collapsible section:
        * Shows AI-generated email + LinkedIn message.
        * Both should be editable in-line with “Save” buttons.
        * Optional toggle: ✅ “Mark this version as sent”
6. Message History per Contact (Optional)
* Beneath generated messages, show:
    * Previous messages sent or saved
    * Date/time, version label (e.g., “Custom Draft A”)
    * Revert or reuse button
