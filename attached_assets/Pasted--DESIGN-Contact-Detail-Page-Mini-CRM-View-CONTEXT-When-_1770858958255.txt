# DESIGN: Contact Detail Page — Mini CRM View

## CONTEXT

When a user clicks a contact name (from All Contacts or from a Job's contact list), they should see a useful detail view for that contact. Currently this page shows "Contact Not Found" (a separate bug to fix first — see REPLIT_FIX_CONTACT_NOT_FOUND.md). This prompt defines what the page SHOULD look like once the routing bug is fixed.

## WHAT THIS PAGE IS FOR

This is a mini CRM view for a single contact. The user needs to:
1. See all info about this person at a glance
2. Generate or view outreach drafts (email + LinkedIn)
3. Track their outreach status with this contact
4. Add personal notes
5. Quickly access the contact's LinkedIn profile and email

- Toggle between Email and LinkedIn views
- "Generate" button calls POST /api/contacts/:id/generate-message
- Generated text appears in an editable textarea so users can tweak it
- "Copy to Clipboard" copies the message
- "Mark as Sent" updates contact_status to "email_sent" or "linkedin_sent" and sets last_contacted_at

If a message has already been generated, show it immediately instead of the generate button.
Store generated messages in `generated_email_message` and `generated_linkedin_message` on recruiter_contacts.

## DATA SOURCE

All data comes from the `recruiter_contacts` table. The existing GET endpoint needs to return:
- All contact fields (name, title, email, linkedin_url, verification_status, etc.)
- The associated job_submission (company_name, job_title) for the "Sourced for" link
- Any existing generated messages

The PATCH /api/contacts/:id endpoint already exists and should handle updates to:
- contact_status
- last_contacted_at
- notes
- email_draft (edited version of generated email)
- linkedin_message (edited version of generated LinkedIn)
