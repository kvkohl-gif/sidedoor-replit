### 📌 WHAT EACH ROUTE SHOULD DO

#### jobSubmissions.js
- POST /api/job-submissions → create a job submission record in Supabase
- GET /api/job-submissions → get all job submissions for a user

#### recruiterContacts.js
- POST /api/contacts → save recruiter contacts for a job submission
- GET /api/contacts/:jobSubmissionId → get contacts for that job

#### outreach.js
- POST /api/outreach/email → generate + log AI email (write to llm_requests + outreach_events)
- POST /api/outreach/linkedin → same for LinkedIn

#### profile.js
- GET /api/profile/:userId → get user_outreach_profiles
- POST /api/profile → update or create profile

#### credits.js
- GET /api/credits/:userId → return current credit balance
- POST /api/credits/spend → deduct credits + log credit_transactions
- POST /api/credits/add → add credits (for testing or admin)

#### subscriptions.js
- GET /api/plans → return subscription_plans
- GET /api/subscription/:userId → return user_subscriptions
- POST /api/subscription/create-checkout-session → create Stripe checkout session

#### webhooks.js
- POST /webhook/stripe
  - handle invoice.payment_succeeded
    → insert into payment_transactions
    → update user_subscriptions
    → add monthly credits to user_credits
  - handle customer.subscription.updated
  - handle customer.subscription.deleted
  - store raw events into stripe_events
