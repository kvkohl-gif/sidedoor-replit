# DESIGN: Outreach Profile Redesign — AI-Assisted + File Upload

## CONTEXT

The Outreach Profile (called "Reference Profile" internally) is the data layer that powers personalized outreach. It's NOT just a form — it's the engine that turns generic AI emails into strategic, reply-worthy messages.

Current state: 7 sections of empty text fields with placeholder text. No AI assistance, no file upload, no guidance on what's most important. Users see a long form and don't know where to start.

## THE REDESIGN

### Principle: Progressive Disclosure + AI Bootstrapping

Instead of showing 7 empty sections, the profile should:
1. Start with the MOST IMPACTFUL fields first (resume upload, then bio)
2. Use AI to analyze uploaded resume and AUTO-FILL other sections
3. Let users review, edit, and expand on AI suggestions
4. Show a completeness indicator so users know how much more they can add

### Profile Completeness Indicator

Add a progress bar at the top of the page:

│  Add your key achievements to improve outreach quality →         │

Scoring:

Show a contextual tip about what to fill next.

### AI Assist Implementation

Each "✨ AI Suggest" button calls a new endpoint:

```
POST /api/outreach-profile/ai-suggest
Body: {
  section: "bio" | "achievements" | "story_hooks",
  context: {
    resumeText: "...",          // from uploaded/pasted resume
    existingBio: "...",         // current bio if any
    existingAchievements: [...], // current achievements if any
  }
}

Response: {
  suggestion: "..." // or array of suggestions for achievements
}
```

This calls OpenAI with a prompt like:

```
You are helping a job seeker build their outreach profile for personalized 
cold emails to recruiters and hiring managers.

Based on their resume, suggest a professional bio that:
- Tells a career narrative, not just lists jobs
- Highlights what makes them unique
- Is 2-3 sentences max
- Sounds human, not corporate

Resume:
{resumeText}
```

For achievements:
```
Extract 3-5 quantifiable wins from this resume. Each should include 
a specific metric (%, $, time, team size). Format as short bullet points.

Resume:
{resumeText}
```

### Resume File Upload Implementation

**Option A (simpler — recommended for now):**
- Accept PDF/DOCX upload on the frontend
- Parse text client-side using pdf.js for PDFs or mammoth.js for DOCX
- Send extracted text to the backend
- Store in a new `resume_text` column on `user_outreach_profiles`
- Optionally store the original file in Supabase Storage and save URL in `resume_url`

**Option B (server-side):**
- Upload file to backend
- Parse with a server-side PDF library
- Store text + file URL

Option A is faster to implement and doesn't require new server dependencies.

### Database Changes

Add these columns to `user_outreach_profiles`:
```sql
ALTER TABLE user_outreach_profiles
ADD COLUMN resume_text text,
ADD COLUMN voice_formality real DEFAULT 0.5,
ADD COLUMN voice_directness real DEFAULT 0.5,
ADD COLUMN voice_length real DEFAULT 0.3,
ADD COLUMN voice_notes text,
ADD COLUMN target_roles text[],
ADD COLUMN target_industries text[],
ADD COLUMN target_company_stage text[],
ADD COLUMN profile_completeness integer DEFAULT 0;
```

Or if array types are complex with Drizzle, store as JSON strings.

## DO NOT
- Do NOT remove any existing sections — just reorder and enhance them
- Do NOT make this depend on the contact/job features — the profile is standalone
- Do NOT change how the outreach message generation reads profile data (just add more data for it to use)
- Do NOT auto-save aggressively — keep the explicit "Save Profile" button but add auto-save for notes
