Task: Restyle “Recent Searches” to match UXpilot table layout
Goal: Replace card rows + status pill with compact table showing Job Title, Company, Date, and Contacts (found + verified), with a “View all job history” link on the right.

Acceptance Criteria
- Section header row:
  - Left: H3 “Recent Searches” + subtext “Your 5 most recent job searches”.
  - Right: ghost button “View all job history →”.
  - Spacing: pt-2 pb-3, container max-w-5xl centered.

- Table (desktop ≥1024px):
  - 4 columns: JOB TITLE, COMPANY, DATE, CONTACTS.
  - Row height ~56–60px; zebra disabled; 1px top divider between rows; no card borders.
  - Contacts cell shows two lines:
      line 1: “{n} found” (semibold, neutral)
      line 2: “{m} verified” (smaller, green-600)
  - Rightmost chevron button per row (→) with hover bg-gray-50; entire row clickable.
  - Remove “Status” pill completely.
  - Truncate long titles with ellipsis; title uses `font-medium text-gray-900`, company `text-gray-600`, date `text-gray-500`.

- Hover/active:
  - Row hover: bg-gray-50.
  - Focus-visible ring on row link: ring-2 ring-blue-500 rounded-md within cell.

- Empty & loading states:
  - If no items: muted card with “No recent searches yet” + primary CTA “Start a new search”.
  - Loading: 5 skeleton rows.

- Mobile (≤640px):
  - Collapse to 2-line rows: Title (bold) + Company; second line: Date • “{found}/{verified}”; chevron at right.

- A11y:
  - Use table semantics (`<table>`, `<thead>`, `<tbody>`, `<th scope="col">`, `<th scope="row">`).
  - Each row is an `<a>` wrapping cells or a button with `aria-label="View results for {title} at {company}"`.

Implementation (React + Tailwind)

```tsx
